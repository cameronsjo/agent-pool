# v0.2 — MCP + State Management

## Context

v0.1 is merged: fsnotify watches postoffice, mail routes to inboxes, experts spawn via `claude -p`, logs capture. Experts currently have no typed tools — they'd need raw file writes to update state, which is fragile and unvalidated.

v0.2 gives experts a **typed MCP tool set** so they can update state, send responses, and recall prior work through validated interfaces. The MCP server runs as a sidecar: the daemon spawns `claude -p --mcp-config <config>`, and Claude Code spawns `agent-pool mcp --pool X --expert Y` as a stdio child process.

## Dependency

Add `github.com/mark3labs/mcp-go` — lightweight, actively maintained Go MCP SDK with stdio transport support. Minimal transitive deps (uuid, cast, jsonschema-go).

## Implementation Steps

### Step 1: State management functions (`internal/expert/state.go`)

Centralize state file read/write/validate operations that MCP tools will call.

**New file:** `internal/expert/state.go`

```go
const MaxStateSize = 50_000 // bytes

func ReadState(expertDir string) (identity, state, errors string, err error)
func WriteState(expertDir, content string) error     // validates non-empty, <= MaxStateSize, atomic write
func AppendError(expertDir, entry string) error       // timestamp-prefixed append to errors.md
func ReadLog(expertDir, taskID string) ([]byte, error) // path-validates taskID
func SearchIndex(expertDir, query string) ([]string, error) // case-insensitive substring match on index.md rows
```

- `ReadState` consolidates `readOptionalFile` calls (already in expert.go)
- `WriteState` uses atomic write (temp file + rename)
- `ReadLog` validates taskID has no path separators (reuse existing `mail.Parse` validation pattern)
- `SearchIndex` parses markdown table rows, skips header

**Tests:** `internal/expert/state_test.go` — happy path, empty/oversized rejection, missing files, path traversal rejection, round-trip consistency.

### Step 2: Mail composition (`internal/mail/compose.go`)

Inverse of `Parse` — builds a mail file string from a `Message`.

**New file:** `internal/mail/compose.go`

```go
func Compose(msg *Message) (string, error) // YAML frontmatter + body, validates required fields
```

**Tests:** `internal/mail/compose_test.go` — round-trip `Compose` → `Parse`, required field validation, special characters in body.

### Step 3: MCP server skeleton (`internal/mcp/server.go`)

Replace the stub `mcp.go` with the server entry point using mcp-go.

**Replace:** `internal/mcp/mcp.go` → `internal/mcp/server.go`

```go
type ServerConfig struct {
    PoolDir    string
    ExpertName string
    Logger     *slog.Logger
}

func Run(ctx context.Context, cfg *ServerConfig) error
```

`Run` creates an mcp-go server, registers expert tools (Step 4), serves on stdio. Logs go to stderr (stdout is the MCP transport).

**Tests:** `internal/mcp/server_test.go` — config validation (empty pool dir, empty expert name).

### Step 4: MCP tool handlers (`internal/mcp/tools.go`)

Six expert tools registered on the server from Step 3.

**New file:** `internal/mcp/tools.go`

| Tool | Input | Calls |
|------|-------|-------|
| `pool_read_state` | none | `expert.ReadState(expertDir)` |
| `pool_update_state` | `content: string` | `expert.WriteState(expertDir, content)` |
| `pool_append_error` | `entry: string` | `expert.AppendError(expertDir, entry)` |
| `pool_send_response` | `to, body, id: string` | `mail.Compose()` → write to `{poolDir}/postoffice/{id}.md` |
| `pool_recall` | `task_id: string` | `expert.ReadLog(expertDir, taskID)` |
| `pool_search_index` | `query: string` | `expert.SearchIndex(expertDir, query)` |

```go
func registerExpertTools(srv *server.MCPServer, cfg *ServerConfig)
```

Each handler resolves `expertDir` as `filepath.Join(cfg.PoolDir, "experts", cfg.ExpertName)`.

**Tests:** `internal/mcp/tools_test.go` — test each handler with `t.TempDir()`, verify filesystem effects. Tests call handlers directly (no stdio transport needed).

### Step 5: MCP config generation + spawn integration

Wire MCP into the daemon's expert spawn path.

**New file:** `internal/mcp/config.go`

```go
type MCPConfig struct {
    MCPServers map[string]MCPServerEntry `json:"mcpServers"`
}

type MCPServerEntry struct {
    Command string   `json:"command"`
    Args    []string `json:"args"`
    Type    string   `json:"type"`
}

func WriteTempConfig(poolDir, expertName string) (path string, err error)
```

Uses `os.Executable()` to find current binary (falls back to `exec.LookPath("agent-pool")`).

**Modify:** `internal/expert/expert.go`
- Add `MCPConfigPath string` to `SpawnConfig`
- In `Spawn()`, if `MCPConfigPath != ""`, append `"--mcp-config", cfg.MCPConfigPath` to args
- Add `AGENT_POOL_DIR` to env vars

**Modify:** `internal/daemon/daemon.go`
- In `processInboxMessage()`, call `mcp.WriteTempConfig()` before spawn, `defer os.Remove()` after

**Tests:**
- `internal/mcp/config_test.go` — verify JSON structure, round-trip parse
- Update `internal/expert/expert_test.go` — verify args include `--mcp-config` when set
- Update `internal/daemon/daemon_test.go` — verify `SpawnConfig.MCPConfigPath` is populated

### Step 6: Hook commands (`internal/hooks/`)

CLI subcommands for Claude Code's hook system.

**New files:** `internal/hooks/flush.go`, `internal/hooks/guard.go`

```go
// flush.go
type FlushConfig struct { PoolDir, ExpertName, TaskID string }
```

**Tests:** `internal/hooks/flush_test.go`, `internal/hooks/guard_test.go`

## Risks

- **mcp-go API surface**: Need to verify actual API matches our assumptions (server creation, tool registration, stdio serving). Read mcp-go source/examples at Step 3 start.
- **Binary self-reference during `go run`**: `os.Executable()` returns a temp binary. Use `make build` for integration testing, not `go run`.
- **Temp file orphaning**: Use `os.CreateTemp` with prefix `agent-pool-mcp-*.json`. Consider startup cleanup sweep (low priority, can defer).
